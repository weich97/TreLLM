# TradeArena Plugin Registry

This directory is a lightweight registry for local and contributed plugins. It
is intentionally small: a plugin should be easy to read, test, and remove if it
does not fit a study.

## Layout

```text
plugins/
  README.md
  local/                  # generated by tradearena new-plugin
  examples/               # curated examples that may be promoted into docs
```

Generated plugins are ignored until a contributor intentionally adds them to a
PR. Keep experimental provider caches outside this directory.

## Suggested Categories

| Category | Example contribution |
| --- | --- |
| `data` | Yahoo Finance sidecar with explicit adjustment policy |
| `analyst` | local Ollama analyst or cached provider adapter |
| `strategy` | SMA crossover, minimum variance, or risk parity allocator |
| `risk` | max drawdown guard, VaR cap, sector concentration guard |
| `simulator` | Almgren-Chriss style market-impact stress proxy |
| `memory` | rolling failure memory with redacted summaries |
| `evaluator` | calibration, concentration, or audit-coverage metric |

## Acceptance Bar

A plugin is ready for review when it has:

- one narrow interface;
- one runnable example or fixture;
- one test that does not require secrets;
- a README section with validation commands;
- no raw provider text, credentials, private holdings, or live-order side
  effects.

See [`docs/plugin_development.md`](../docs/plugin_development.md) for the
interface contract and scaffold command.
